Detailed Comparison

The same benchmark run on the same data across the whole history of ClickHouse versions, to show how performance evolved over the years. It combines several datasets and query sets: MgBench, the Star Schema Benchmark, ClickBench (the "hits" web-analytics dataset), the UK property price paid dataset, the airline OnTime dataset, the New York taxi rides benchmark, a coffee-shop sales dataset, TPC-H, TPC-DS, and the Join Order Benchmark (JOB).

Every version is packaged as a Docker image and runs the identical set of queries over the identical (Native-format) data. Versions with no published image — down to the earliest 2013–2016 snapshots — are reconstructed and built from source (see build-from-source). Very old versions predate SELECT version() and report their revision as 0.0.<revision>.

"Cold Run" is the first run of a query (cold OS/page cache dropped between queries); "Hot Run" is the fastest of the subsequent runs. A query that failed or is unsupported on a version is shown as ☠ and penalised in the relative-time summary. Per-dataset load time (sum of the load of that dataset's tables) and on-disk data size are shown in the dataset header rows.
